In 26.1 most recipe types have been combined into one recipe using Triggers, Conditions, Results and Options.

This recipe type is called "inworldrecipes:world_recipe"

# Triggers
Triggers are what cause the recipe to run, multiple triggers can be used and all triggers must be satisfied for the recipe to run.

## Click Type
Can be "left" or "right" clicking a block in world
```js
    "click_type": "left"
```

## Target Block
The block that must be targeted for recipe to trigger, pairs well with click type. Uses either
```js
    "target_block": { "Name": "minecraft:diamond_block" }
```
or
```js
    "target_block": "#minecraft:dirt"
```

## Lightning Strike (NYI)
Triggers when a lightning strike happens within a 3 block range of the target area. If missing uses false
```js
"lightning_strike": true
```

## Standing On
Triggers when the player is standing on a certain block. Uses either
```js
    "standing_on_block": { "Name": "minecraft:diamond_block" }
```
or
```js
    "standing_on_block": "#minecraft:dirt"
```

# Conditions
Conditions are what must be true for the recipe to run, multiple conditions can be used and all conditions must be satisfied for the recipe to run.

## Held Item
Player must be holding a certain item, uses either. Uses SizedIngredient format
```js
   {"held_item": {"ingredient": "minecraft:diamond", "count": 1}}
```

## Ignore Block State
Ignores the block state of the target block or standing on block, useful for blocks with many states like stairs or beds. Uses true or false
```js
    "ignore_block_state": true
```

## Dropped Items
Must have these items in the area when the recipe is triggered, uses a List<SizedIngredient>
```js
    "dropped_items": [{"ingredient": "minecraft:dirt","count": 3}]
```

## Inventory Items
Must have these items in the inventory when the recipe is triggered, uses a List<SizedIngredient>
```js
    "inventory_items": [{"ingredient": "minecraft:dirt","count": 3}]
```

## Weather
Must be a certain weather when the recipe is triggered, can be "clear", "rain", "thunder", "snow". If missing uses nother and any weather will work. Note the weather set to rain will also work for thunder but thunder recipes will not work in rain
```js
    "weather": "rain"
```

# Results 
Results are what happens when the recipe is triggered and all conditions are satisfied, multiple results can be used and all results will happen when the recipe is triggered.

## Chance Results
Chance results have a chance to happen when the recipe is triggered, uses a List<ChanceResult> 
```js
      "results": [
        {
          "item": {
            "id": "minecraft:acacia_log",
            "count": 2
          },
          "chance": 0.5
        },
        {
          "item": {
            "id": "minecraft:emerald",
            "count": 7
          },
          "chance": 1
        }
      ]
```

## Damage Held Item
Damages the held item by 1 durability, if the item breaks it will be removed from the player's inventory. Uses true or false. If missing uses false
```js
    "damage_held_item": true
```

## Consume Held Item
If the Held Item is consumed it will be taken with the same count as the recipe requires, if the count is not specified it will consume 1 of the held item. Uses true or false. If missing uses false
```js
    "consume_held_item": true
```

## Pop Items
If item results are popped they are placed in world at the location of the trigger. If false they are placed directly into the player's inventory, if missing uses false
```js
    "pop_items": true
```

## Output Blockstate 
Changes the blockstate of the target block or standing on block to the specified blockstate
```js
      "block_state": {
        "Name": "minecraft:obsidian"
      }
```

## Consume Inventory Items
If the Inventory Items are consumed they will be taken with the same count as the recipe requires, if the count is not specified it will consume 1 of each inventory item. Uses true or false. If missing uses false
```js
    "consume_inventory_items": true
```

## Consume Dropped Items
If the Dropped Items are consumed they will be taken with the same count as the recipe requires, if the count is not specified it will consume 1 of each dropped item. Uses true or false. If missing uses false
```js
    "consume_dropped_items": true
```

# Options
Options are extra settings for the recipes eg showing the recipe in jei or making the recipe just visible in jei and not actually do anthing. Useful for showing methods from other mods that use world related mechanics like right clicking a block with an item or standing on a block. Allows you to show these in jei without effecting the mods own behavior.

## Show in JEI
Shows the recipe in jei, uses true or false. If missing uses true
```js
    "show_in_jei": false
```

## Only Visual Recipe
Shows the recipe in JEI, but it doesn't actually do anything, useful for showing methods from other mods that use world related mechanics like right clicking a block with an item or standing on a block. Uses true or false. If missing uses false
```js
    "only_visual_recipe": true
```


# Example Recipes
```json
{
  "type": "inworldrecipes:world_recipe",
  "triggers": [
    {
      "standing_on_block": { "Name": "minecraft:acacia_log" },
      "click_type": "right"
    }
  ],
  "conditions": [
    {
      "held_item": {
        "ingredient": "minecraft:emerald"
      },
      "ignore_block_state": true,
      "dropped_items": [
        {
          "ingredient": "minecraft:dirt",
          "count": 3
        }
      ],
      "inventory_items": [
        {
          "ingredient": "minecraft:oak_log"
        }
      ]
    }
  ],
  "results": [
    {
      "consume_held_item": false,
      "consume_inventory_items": true,
      "consume_dropped_items": true,
      "results": [
        {
          "item": {
            "id": "minecraft:diamond_block",
            "count": 4
          }
        },
        {
          "item": {
            "id": "minecraft:iron_block",
            "count": 1
          }
        },
        {
          "item": {
            "id": "minecraft:emerald_block",
            "count": 1
          }
        }
      ]
    }
  ]
}
```
